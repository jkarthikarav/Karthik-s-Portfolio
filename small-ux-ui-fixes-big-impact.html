Design Process

While testing Buoy’s health chatbot for a minor real-life symptom, I noticed a key usability flaw: once a symptom is added, there’s no way to edit it; attempting to remove one symptom causes the entire list to be cleared, with no option to recover or undo the action. This flaw was so frustrating that I had to stop midway through the current user flow. My redesign focuses on this early step, as patients facing this issue might become frustrated and abandon the process altogether.

This behavior breaks at least three of Nielsen’s 10 Usability Heuristics:
1. User Control and Freedom
2. Recognition Rather Than Recall
3. Help Users Recognize, Diagnose, and Recover from Errors

Mapping the Current Chatbot Flow

Design Recommendations

Key Friction Step

Here, users cannot remove a single symptom (e.g., "headache") without causing the deletion of the entire list, and there's no option to directly edit symptoms without restarting at the symptom input step.

Fix 01: Allow Symptom Editing

Allow users to edit symptoms after they've selected.

→
Impact: Improves user control and reduces frustration, helping users interact with the system more intuitively.
Fix 02: Enable Selective Removal

Allow users to remove individual symptoms without clearing the entire list.

→
Impact: Restores user control, reduces frustration, and aligns with user mental models for conversational AI tools.
