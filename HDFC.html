While institutions race toward mobile apps and immersive tech, many users still rely on browser-based banking for high-trust transactions. Is HDFC's user interface discouraging usage?

Project Context

India’s largest private bank uses a netbanking system that feels out of step with user expectations in 2025. As a long-time HDFC customer from a family of bankers, I decided to investigate the usability gaps in their web interface and explore how better design, without a full UI overhaul, could deliver real value to users. Despite rapid shifts to mobile and VR-first experiences, millions still rely on desktop netbanking, especially for high-value transactions.

This redesign improves usability, clarity, and interaction patterns of HDFC’s platform using the existing brand system (i.e., no new fonts, colors, or UI kits) just better interface design and information structure.

User Journey: Check if Beneficiary is added and Make a Transaction

I used user journeys to map out task flows and decision-making processes, helping identify friction points and guide a more intuitive redesign.

Step 0: Arrives at Home (Account Summary).

Step 1: Clicks Transfer Funds in main menu.

Step 2: Clicks Requests in sidebar.

Step 3: Selects View/Delete Beneficiaries.

Step 4: If beneficiary is saved, either Views Beneficiary or goes back to Transact and initiates a transfer.

What I Learned

The goal of this redesign was to modernize a legacy netbanking experience without discarding its visual identity or disrupting long-time users. Working within HDFC’s existing UI system challenged me to focus on layout logic, visual density, and information flow rather than flashy changes. This sharpened my systems thinking, especially in reorganizing features like Beneficiary Management and Scheduled Transfers into more intuitive sections. The project also helped me apply information architecture principles to legacy content and workflows while respecting technical constraints.

Key Takeaway: Prototyping inside real projects accelerates adoption

You don’t need to overhaul a UI kit to make a legacy platform easier to use. Reworking structure, hierarchy, and logic can go a long way in improving experience and building trust.
